# STOCK.md - BrainScience 주식 분석 가이드라인 (v3.5)

> 목적: "읽기 쉬운 설명문"이 아니라 "한눈에 보고 바로 판단 가능한 실행 문서"를 만든다.
> 원칙: 실시간 데이터 검증, 뉴스와 데이터의 교차검증, ASCII 시각화 적극 활용, 위험도 등급화, 도메인 심층 조사, 전역 스카우팅, 실행 전략 구체화.

---

## 0. 이 문서의 기준

- 모든 주식 리포트는 이 가이드라인을 강제 준수한다.
- 특히 아래 10가지는 절대 누락 금지다.
  - 현재가 / 역사적 최저가 / 역사적 최고가 표
  - 기간별 저가/고가 표
  - ASCII 가격 그래프 및 비교 그래프
  - 한눈에 보는 핵심 요약 대시보드
  - 위험도 등급 + 위험 사유 요약
  - 모든 핵심 판단에 대한 데이터 근거
  - 뉴스와 데이터의 연결 검증
  - 넓은 범위의 유망 후보 발굴 및 숨은 종목 스카우팅
  - 왜 사야 하는지 / 왜 아직 사면 안 되는지에 대한 종합 인사이트
  - 무엇을 언제 어떻게 사고, 언제 어떻게 파는지에 대한 실행 전략

## Ⅰ. 최우선 작성 원칙

1. **실시간 검증 우선**
   - 반드시 웹 검색으로 현재가, 52주 저가/고가, 시가총액, 실적 일정, 최근 뉴스, 금리/정책/섹터 흐름을 확인한다.
   - 날짜는 반드시 절대 날짜로 쓴다. 예: `2026-03-19`.

2. **가격만 말하지 말고 맥락까지 연결**
   - 가격 변화 원인을 `실적`, `가이던스`, `수급`, `정책`, `기술 마일스톤`, `경쟁`, `밸류에이션`으로 분해해서 설명한다.

3. **도메인 분석을 충분히 깊게**
   - 기업 분석은 반드시 산업 구조, 기술 우위, 공급망, 고객, 경쟁사, 규제, 자본조달, 수익성 구조까지 포함한다.
   - "좋아 보인다" 수준 금지. 왜 그런지 숫자와 구조로 설명한다.

4. **실전 투자 관점 필수**
   - 리포트는 반드시 `매수 전략`, `비중 전략`, `추가 매수 조건`, `손절/축소 조건`, `익절/청산 조건`을 포함한다.

5. **ASCII 시각화 적극 사용**
   - 표로 끝내지 않는다.
   - 가격 위치, 리스크/리턴, 경쟁 구도, 시나리오, 의사결정 트리를 ASCII로 반드시 시각화한다.

6. **길게 쓰지 말고 압축해서 쓴다**
   - 문단형 장문 금지.
   - 각 섹션은 `결론 1줄 + 핵심 사유 3개 이내 + 표/그래프` 중심으로 작성한다.

7. **위험도는 반드시 등급화한다**
   - 단순히 "리스크가 있다"로 끝내지 않는다.
   - `낮음 / 보통 / 높음 / 매우 높음`으로 명확히 표시하고, 왜 그런지 핵심 사유를 짧게 병기한다.

8. **종목은 주어진 것만 보지 말고 직접 발굴한다**
   - 리포트 작성 전에 반드시 `대형주`, `중소형주`, `신규상장`, `해외시장`, `정책 수혜주`, `공급망 숨은 수혜주`를 넓게 스캔한다.
   - 사용자가 먼저 언급하지 않은 종목이라도 더 유망하면 후보에 포함한다.

9. **숨은 수혜주를 찾는다**
   - 메인 플레이어만 보지 않는다.
   - 장비, 소재, 부품, 인증, 전력, 냉각, 테스트, 소프트웨어, 데이터, 유통, 계약 수혜 기업까지 역으로 추적한다.

10. **모든 판단은 데이터 근거가 있어야 한다**
   - `좋다`, `유망하다`, `위험하다` 같은 표현만 쓰지 않는다.
   - 반드시 가격, 밸류에이션, 성장률, 수주, 점유율, 마진, 현금흐름, 고객 수, 승인 일정 같은 수치 근거를 붙인다.

11. **뉴스는 단독 근거가 아니라 검증 대상이다**
   - 뉴스가 나왔다고 바로 결론 내리지 않는다.
   - 뉴스가 실제 매출, 현금흐름, 주문, 규제 변화, 가격 행동으로 이어지는지 확인한다.

12. **판단과 근거를 1:1로 매핑한다**
   - 최종 액션, 위험도, 목표가, 촉매 판단마다 `왜 그렇게 보는지`를 표 형태로 남긴다.

13. **출처 품질이 낮으면 버린다**
   - 저품질 기사, 선정적 제목, 출처 불명 루머, 재가공 블로그성 글은 분석 근거로 쓰지 않는다.
   - 1차 소스와 상위급 매체가 아니면 반드시 추가 검증 후에만 제한적으로 쓴다.

14. **세계 시장과 현재 국면을 먼저 본다**
   - 개별 종목보다 먼저 금리, 유동성, 정책, 환율, 원자재, 지정학, 섹터 순환, 국가별 자본지출 흐름을 본다.
   - 종목 판단은 항상 `세계 -> 국가 -> 산업 -> 기업` 순서로 내린다.

15. **도메인 이해 없는 뉴스 해석 금지**
   - 산업 구조를 모르면 뉴스 해석을 멈춘다.
   - 뉴스는 도메인 이해를 보조하는 재료일 뿐, 산업의 본질을 대체할 수 없다.

16. **표와 ASCII 그래프를 과하게 쓸 정도로 적극 활용한다**
   - 문단보다 표를 우선한다.
   - 가격, 위험도, 경쟁 구도, 시나리오, 액션 플랜은 가능한 한 모두 표와 ASCII 그래프로 표현한다.
   - 리포트 하나에 표 8개 이상, ASCII 그래프 5개 이상을 기본 기준으로 한다.

17. **섹션별 시각화 의무를 둔다**
   - 가격 섹션: 표 2개 + ASCII 그래프 2개 이상
   - 뉴스/촉매 섹션: 표 1개 이상
   - 경쟁/위험도 섹션: 표 1개 + ASCII 그래프 1개 이상
   - 목표가/전략 섹션: 표 2개 + ASCII 그래프 1개 이상

## Ⅲ. 리포트 상단 템플릿

### # [종목명/티커] 분석 및 트레이딩 마스터플랜

#### 핵심 인사이트 (3줄 요약)
> 1. **본질 (What)**: 이 회사가 실제로 무엇을 팔고, 왜 중요한가.
> 2. **판단 (Why now)**: 지금 이 시점에 봐야 하는 이유와 핵심 촉매는 무엇인가.
> 3. **액션 (How)**: 지금 사야 하는지, 기다려야 하는지, 팔아야 하는지.

### 한눈에 보는 투자 대시보드 (필수)

리포트 최상단에 아래 표를 반드시 넣는다.

| 항목 | 결론 | 사유 |
|---|---|---|
| 최종 액션 | 매수 / 분할매수 / 관망 / 축소 / 매도 | 가장 핵심 근거 1줄 |
| 투자 매력도 | A / B / C / D | 왜 지금 매력적인지 1줄 |
| 위험도 | 낮음 / 보통 / 높음 / 매우 높음 | 왜 위험한지 1줄 |
| 시간축 | 단기 / 중기 / 장기 | 어느 기간에서 유리한지 1줄 |
| 핵심 촉매 | 상 / 중 / 하 | 가장 중요한 이벤트 1줄 |
| 핵심 리스크 | 상 / 중 / 하 | 가장 중요한 깨짐 조건 1줄 |

### 투자 판단 ASCII 신호등 (필수)

```text
[한눈에 보는 투자 신호]

매력도   : [A]
위험도   : [높음]
타이밍   : [분할매수]
시간축   : [중기]
촉매강도 : [상]

요약:
- 사는 이유: 수요 성장 + 기술 우위 + 실적 가시성
- 조심할 이유: 밸류 부담 + 변동성 + 고객 집중
```

### 판단 근거 요약표 (필수)

최상단에 아래 표를 추가한다.

| 판단 | 결론 | 데이터 근거 | 뉴스/이벤트 근거 |
|---|---|---|---|
| 최종 액션 | 분할매수 | 매출 성장률, 52주 위치, 밸류 지표 | 실적 발표, 수주 공시 |
| 위험도 | 높음 | 변동성, 부채, 고객 집중도 | 규제 이슈, 경쟁 뉴스 |
| 목표가 | Base $000 | peer 멀티플, 실적 추정 | 산업 수요 변화 |

### 판단-근거 ASCII 체인 (필수)

```text
[판단 근거 체인]

뉴스/공시
   |
   v
데이터 확인
   |
   v
해석
   |
   v
액션

예:
수주 발표 -> 주문잔고 증가 확인 -> 매출 가시성 상승 -> 분할매수
```

### 상단 요약 카드 표 (강력 권장)

```text
+----------------+----------------------+----------------------+
| 액션           | 분할매수             | 근거: 눌림목/실적    |
| 위험도         | 높음                 | 근거: 변동성/밸류    |
| 가격 위치      | 52주 상단 68%        | ATH 대비 -18%        |
| 핵심 촉매      | 다음 실적/수주 공시  | 3개월 내             |
+----------------+----------------------+----------------------+
```

## Ⅳ. 세계 시장 및 현재 국면 (신규 핵심)

개별 종목에 들어가기 전에 반드시 지금 시장의 큰 판을 먼저 요약한다.
종목은 혼자 움직이지 않는다. 금리, 유동성, 정책, 환율, 원자재, 지정학, 섹터 순환 속에서 움직인다.

### 1. 상단 매크로 요약표 (필수)

| 축 | 현재 판단 | 왜 중요한가 |
|---|---|---|
| 금리/유동성 | 완화 / 중립 / 긴축 | 성장주 밸류에이션에 직접 영향 |
| 달러/환율 | 강세 / 중립 / 약세 | 수출/수입 기업 및 글로벌 자금 영향 |
| 에너지/원자재 | 상승 / 안정 / 하락 | 제조업과 인프라 원가 영향 |
| 정책/규제 | 우호 / 중립 / 역풍 | 산업 재평가 속도 결정 |
| 섹터 순환 | AI / 반도체 / 바이오 / 산업재 등 | 자금 유입 방향 확인 |
| 지정학 | 낮음 / 중간 / 높음 | 공급망과 리스크 프리미엄 반영 |

### 2. 세계 -> 국가 -> 산업 -> 기업 분석 체인 (필수)

```text
[탑다운 분석 체인]

세계 거시
   |
   v
국가 정책 / 환율 / 금리
   |
   v
산업 구조 / 수요 / capex
   |
   v
기업 경쟁력 / 실적 / 가격
   |
   v
매수 / 관망 / 매도
```

### 3. 현재 국면 한줄 요약 (필수)

| 항목 | 결론 | 근거 |
|---|---|---|
| 시장 국면 | 리스크온 / 중립 / 리스크오프 | 금리, 유동성, 지수 흐름 |
| 성장주 환경 | 우호 / 중립 / 불리 | 멀티플 확장/축소 |
| 해당 섹터 환경 | 우호 / 중립 / 불리 | 수요, 공급, 정책, capex |

## Ⅴ. 판단 근거 규칙 (신규 핵심)

이 섹션은 모든 리포트에 공통으로 적용된다.
핵심은 "뉴스를 읽고 의견을 쓰는 것"이 아니라 "데이터로 검증하고 결론을 내리는 것"이다.

### 1. 판단 근거 표준 형식 (필수)

모든 중요한 주장에 아래 형식을 붙인다.

| 주장 | 결론 | 근거 데이터 | 뉴스/이벤트 | 최종 해석 |
|---|---|---|---|---|
| 성장성 | 높음 | 매출 성장률, 수주잔고, 고객 증가 | 신규 계약, 가이던스 | 실제 실적 연결 가능 |
| 밸류 부담 | 높음 | PER, PSR, EV/EBITDA | 동종업계 재평가 뉴스 | 기대 선반영 가능성 |
| 재무 위험 | 낮음 | 순현금, FCF, 부채 만기 | 자금조달 뉴스 | 생존력 양호 |

### 2. 데이터 우선순위 (필수)

아래 순서대로 신뢰한다.

1. 기업 공시 / 실적 발표 / IR 자료
2. 정부 문서 / 규제 문서 / 승인 일정
3. 거래소 공시 / 공식 발표
4. 1차 인터뷰 / 컨퍼런스 콜
5. 주요 언론 보도
6. 시장 루머 / 2차 재가공 콘텐츠

### 2-1. 출처 등급제 (필수)

| 등급 | 허용 여부 | 예시 | 사용 방식 |
|---|---|---|---|
| S | 적극 사용 | 공시, 실적 발표, 정부/규제 문서, 거래소 공시 | 핵심 근거 가능 |
| A | 사용 가능 | FT, Bloomberg, Reuters, WSJ 등 상위권 매체 | 보조 근거 가능 |
| B | 제한적 사용 | 업계 전문지, 기업 인터뷰 기사 | S/A와 교차검증 필요 |
| C | 원칙적 배제 | 클릭 유도형 기사, 출처 불명 블로그, 재가공 콘텐츠 | 근거 사용 금지 |
| D | 금지 | SNS 루머, 익명 텔레그램/커뮤니티 소문 | 사용 금지 |

### 2-2. 금지 소스 규칙 (필수)

다음은 분석 근거로 쓰지 않는다.

- 출처가 없는 카더라 기사
- 제목만 자극적인 요약 사이트
- 원문 확인이 안 되는 2차 재가공 글
- 근거 수치가 없는 유튜브식 주장
- SNS 루머, 커뮤니티 찌라시

### 3. 뉴스 검증 규칙 (필수)

뉴스는 아래 4가지를 반드시 본다.

| 체크 항목 | 질문 |
|---|---|
| 사실성 | 공식 확인이 되었는가 |
| 금액성 | 실제 매출/이익/수주 규모로 이어지는가 |
| 지속성 | 하루짜리 뉴스인가, 장기 추세인가 |
| 가격 반영 | 이미 주가가 선반영했는가 |

### 4. 뉴스 신뢰도 표 (필수)

| 뉴스/이벤트 | 신뢰도 | 데이터 연결 | 투자 의미 |
|---|---|---|---|
| 기업 공시 | 매우 높음 | 직접 연결 | 강함 |
| 실적 발표 | 매우 높음 | 직접 연결 | 강함 |
| 정부 발표 | 높음 | 간접/직접 연결 | 중~강 |
| 주요 언론 단독 | 중간 | 확인 필요 | 중간 |
| 루머/SNS | 낮음 | 불명확 | 약함 |

### 5. 숫자 없는 판단 금지 규칙 (필수)

다음 표현은 숫자 없이 쓰지 않는다.

- 성장성이 높다
- 밸류가 비싸다
- 위험도가 높다
- 실적이 좋다
- 수요가 강하다
- 모멘텀이 있다

각 표현 뒤에는 반드시 숫자 또는 검증 데이터가 붙어야 한다.

예:

- 잘못된 예: "성장성이 높다"
- 올바른 예: "최근 4개 분기 매출 성장률이 28%, 31%, 35%, 39%로 가속 중이어서 성장성이 높다"

## Ⅵ. 전역 스카우팅 및 숨은 후보 발굴 (신규 핵심)

이 섹션은 "이미 아는 종목 분석" 이전에 반드시 수행한다.
분석 대상은 좁게 잡지 않는다. 미국만 보지 않고, 한국, 일본, 유럽, 신흥시장까지 넓게 본다.

### 1. 탐색 범위 (필수)

최소 아래 풀을 동시에 본다.

| 탐색 축 | 범위 | 목적 |
|---|---|---|
| 시가총액 | 대형 / 중형 / 소형 / 마이크로캡 | 아직 덜 알려진 후보 발굴 |
| 시장 | 미국 / 한국 / 일본 / 유럽 / 기타 | 편향 제거 |
| 단계 | 상장 초기 / 턴어라운드 / 실적 가속 / 흑자 전환 전 | 초기 기회 포착 |
| 산업 위치 | 메인 플레이어 / 공급망 / 장비 / 소재 / 소프트웨어 / 서비스 | 숨은 수혜주 발굴 |
| 촉매 | 실적 / 규제 / 허가 / 수주 / 증설 / 특허 / 계약 | 선행 신호 포착 |

### 2. 반드시 찾아야 하는 후보 유형 (필수)

- 시장이 이미 아는 대표주
- 대표주 뒤에 숨어 있는 공급망 수혜주
- 매출은 아직 작지만 수주나 계약이 먼저 잡히는 기업
- 규제 승인이나 정책 변화로 급격히 재평가될 기업
- 대기업과 계약했지만 아직 시장이 숫자로 반영하지 않은 기업
- 산업 사이클이 반전되는데 주가는 아직 바닥권인 기업
- 실적은 아직 약하지만 주문잔고, 고객 증가, 생산능력 증설이 먼저 보이는 기업

### 3. 스카우팅 데이터 소스 (필수)

반드시 뉴스 기사만 보지 말고 아래 1차 소스를 우선 확인한다.

| 소스 유형 | 무엇을 찾는가 | 왜 중요한가 |
|---|---|---|
| 공시 / 규제 filing | 신규 계약, 자금조달, 리스크, 내부자 변화 | 가장 원본에 가까움 |
| 실적 발표 / IR 자료 | 가이던스, 고객, 주문잔고, capex | 숫자 검증 가능 |
| 정부 계약 / 정책 공고 | 수주, 보조금, 정책 수혜 | 선행 수요 파악 |
| 특허 / 기술 공시 | 기술 선점, IP 이전, 상용화 준비 | 아직 가격 반영 전 신호 |
| 승인 일정 / 허가 캘린더 | 바이오, 국방, 에너지, 인프라 촉매 | 이벤트 드리븐 기회 |
| 거래소 공시 시스템 | 국가별 조기 공시 탐색 | 미국 밖 기회 탐색 |

### 4. 아이디어 발굴 질문 (필수)

매번 아래 질문으로 좁혀간다.

- 지금 시장의 합의가 어디에 몰려 있는가
- 그 합의 뒤에서 돈을 버는 숨은 기업은 누구인가
- 아직 매출은 작지만 수주와 고객이 먼저 붙는 곳은 어디인가
- 정책, 허가, 규제 변화로 가장 먼저 이익을 보는 곳은 어디인가
- 대기업의 capex 증가가 누구 주문으로 연결되는가
- 시장이 `테마`만 보는데 실제 `현금흐름`이 생기는 곳은 어디인가

### 5. 전역 후보 발굴 표 (필수)

최소 5개 이상 후보를 먼저 뽑고, 그중에서 리포트 본문 분석 대상을 정한다.

| 후보 | 국가 | 시총 구간 | 테마 | 아직 덜 알려진 이유 | 먼저 봐야 하는 신호 |
|---|---|---|---|---|---|
| A | 미국 | 중형 | AI 인프라 | 대형주 뒤에 가려짐 | capex 수주 증가 |
| B | 한국 | 소형 | 전력/냉각 | 거래량 작음 | 고객사 확대 |
| C | 일본 | 소형 | 로봇/부품 | 내수주로 오해 | 수출 증가 |
| D | 유럽 | 중형 | 국방/전력 | 미국 투자자 관심 낮음 | 정책 예산 증가 |
| E | 기타 | 소형 | 바이오/소재 | 아직 허가 전 | 승인 일정 |

### 7. 시장 편향 제거 규칙 (필수)

- 최근 급등주만 추격하지 않는다
- 시총이 작다는 이유만으로 배제하지 않되, 유동성과 재무 위험은 별도 체크한다

## ⅩⅣ. 최종 결론 형식

결론은 의견이 아니라 근거 압축이어야 한다.

결론은 반드시 아래 6문장 안팎으로 정리한다.

1. 지금 이 종목을 한 문장으로 정의
2. 현재 가격이 싼지 비싼지
3. 지금 당장 살지, 기다릴지, 피할지
4. 가장 중요한 상승 촉매 1~2개
5. 가장 중요한 하락 리스크 1~2개
6. 최종 액션: `매수 / 분할매수 / 관망 / 비중축소 / 매도`

## ⅩⅤ. 요약 작성 규칙 (신규 필수)

최종 문서의 모든 큰 섹션은 아래 규칙을 지킨다.

- 각 섹션 첫 줄에 `한줄 결론`을 쓴다.
- 각 섹션 설명은 3개 포인트 이내로 압축한다.
- 가능하면 문단 대신 표를 우선한다.
- 긴 설명이 필요하면 표 아래 `해석:` 2~3줄로 끝낸다.
- "좋다", "나쁘다" 같은 추상 표현만 쓰지 말고 반드시 이유를 붙인다.
- 이유는 가능하면 숫자와 데이터로 쓴다.
- 독자가 10초 안에 `액션`, `위험도`, `가격 위치`를 파악할 수 있어야 한다.
- 표와 ASCII 그래프만 훑어도 핵심이 이해되어야 한다.

### 권장 형식

| 항목 | 결론 | 사유 |
|---|---|---|
| 매수 판단 | 분할매수 | 밸류 부담은 있으나 수요 가시성 높음 |
| 위험도 | 높음 | 변동성 크고 실적 민감도 큼 |
| 가격 위치 | 중상단 | 52주 밴드 상단 70% 수준 |

- [ ] 뉴스가 실제 데이터와 연결되는지 검증했는가
- [ ] 루머/기사와 공시/실적을 구분했는가
- [ ] 현재가, ATH, ATL, 52주 고가, 52주 저가를 모두 확인했는가
- [ ] 기간별 저가/고가 표가 들어갔는가
- [ ] 한눈에 보는 투자 대시보드가 문서 최상단에 있는가
- [ ] 종합 위험도와 사유가 한 줄로 정리되었는가
- [ ] 위험 항목별 등급, 사유, 확인 신호, 대응이 들어갔는가
- [ ] ASCII 가격 그래프가 최소 3개 이상 들어갔는가
- [ ] 최근 뉴스와 가격 반응을 연결했는가
- [ ] 산업/기술/재무/경쟁/규제를 모두 조사했는가
- [ ] 왜 오를 수 있는지와 왜 틀릴 수 있는지를 둘 다 썼는가
- [ ] 매수 가격, 비중, 추가 매수 조건을 썼는가
- [ ] 매도 가격, 손절 조건, thesis break 조건을 썼는가
- [ ] 최종 액션이 한 줄로 명확한가

## ⅩⅦ. 품질 기준

다음 중 하나라도 빠지면 미완성 리포트로 본다.

- 표가 부족하고 문단 위주로 서술함
- ASCII 그래프가 형식적으로 1~2개만 들어감
- 가격/위험도/전략이 시각화 없이 서술로만 끝남
- 출처 수준이 낮고 검증이 없음
- 세계 시장/거시/섹터 국면이 빠져 있음
- 도메인 이해 없이 뉴스 나열만 함
- 주어진 종목만 분석하고 새로운 후보 탐색이 없음
- 메인 플레이어만 보고 숨은 수혜주 추적이 없음
- 미국 시장만 보고 글로벌 탐색이 없음
- 뉴스는 있는데 데이터 검증이 없음
- 결론은 있는데 근거 표가 없음
- 위험도는 있는데 수치 근거가 없음
- 핵심 요약 대시보드가 없음
- 위험도 등급은 있는데 왜 그런지 사유가 없음
- 사유는 있는데 대응 전략이 없음
- 가격 데이터는 있는데 기간별 범위가 없음
- 표는 있는데 ASCII 그래프가 없음
- 뉴스는 있는데 도메인 분석이 얕음
- 목표가는 있는데 근거가 약함
- 매수 전략은 있는데 매도 전략이 없음
- "좋다/유망하다"는 말은 있는데 언제 어떻게 행동할지가 없음

## ⅩⅧ. 최종 작성 원칙 한 줄 요약

> 좋은 주식 리포트는 저품질 뉴스에 휘둘리지 않고, 세계 시장과 도메인 본질을 먼저 본 뒤, 검증된 데이터와 출처로 가격 위치와 위험도와 액션을 한눈에 보여주는 실행 문서여야 한다.
